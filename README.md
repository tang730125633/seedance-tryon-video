# Seedance 真人视频 · 视频换装 & 换背景

本仓库只做两件事：给真人短视频**换装**和**换背景**。

- **换装**：保持原视频里的人物、脸、动作、镜头不变，只把身上的衣服换成一张衣服参考图里的款式。
- **换背景**：在换装成片的基础上，把背景替换成指定的背景图（例如花墙、门店、活动场景）。

整个流程由 **Codex（AI agent）驱动脚本批量执行**，实际生成走 RunningHub 的两个 AI App。

---

## 当前稳定链路：RunningHub 两段式

不要追求"一步到位"。目前最稳、可复现的做法是**先换装、再换背景**两段式。

### 第 1 段 · 换装

| 项 | 值 |
|---|---|
| AI App | `视频换装V2 Animate Wan2.2=B站艾橘溪` |
| WebApp ID | `1969605116187844610` |
| 节点 | `363.video`=原视频 · `373.image`=衣服参考图 · `362.value`=秒数 · `358.value`=宽 · `359.value`=高 · `372.text`=替换目标 |
| 推荐参数 | 尺寸 `576 x 1024`，替换目标填 **`Clothes`**（不要写复杂中文服装描述），生成后再放大到原尺寸 |

> 关键：`372.text` 写 `Clothes` 比写"蓝白格纹荷叶边白蝴蝶结连衣裙"这类详细描述更稳。详细提示词容易让模型保留原衣服或理解错领口/肩部结构。衣服参考图要干净，别带手机 UI、黑边或杂乱背景。

### 第 2 段 · 换背景

| 项 | 值 |
|---|---|
| AI App | `【c0120】VideoRefusion 动态替换视频背景` |
| WebApp ID | `1986353521488523266` |
| 节点 | `352.video`=换装后的成片 · `318.image`=背景图 · `339.int`=视频时长 |

> 关键：背景阶段很慢，容易长时间卡在 `RUNNING` 但不一定失败，建议用并行续跑脚本而不是串行等待。背景阶段可能再次影响脸部和边缘，成片后要做质检。

## 标准作业流程

1. **整理客户素材**——每个客户一个目录，固定三件套：
   ```text
   客户01/
     人物视频/          # 原视频（要换装的人）
     衣服图或参考视频/   # 目标衣服
     背景图/            # 目标背景
   ```
2. **扫描 + 生成 manifest**
   ```bash
   python3 "90-AI脚本与代码/scripts/scan_tryon_clients.py"        --root "换装视频测试/客户素材待处理-XXX"
   python3 "90-AI脚本与代码/scripts/build_tryon_client_manifest.py" \
       --root "换装视频测试/客户素材待处理-XXX" \
       --out  "换装视频测试/客户素材待处理-XXX/ready_clients_manifest.json"
   ```
3. **批量换装 + 换背景**（并行续跑，避免被慢任务卡死）
   ```bash
   python3 "90-AI脚本与代码/scripts/run_tryon_bg_from_manifest.py" --manifest <上面的 manifest>
   ```
4. **质检**——生成预览页 HTML + 质检拼图，人工挑选后放进交付目录。
