## 关键经验

### 1. 不要默认把长视频按 6 秒短测跑

之前的成功链路默认参数是：

- `seconds=6`
- `width=576`
- `height=1024`
- `target=Clothes`

这适合短测，但如果用户给的是 15 秒/10 秒原视频，必须先确认是否要保留原时长。用户这次明确要求长版，所以 6 秒版不能当最终交付。

### 2. 换装工作流可以跑长版

`视频换装V2 Animate Wan2.2=B站艾橘溪`：

- WebApp ID：`1969605116187844610`
- 客户01 15 秒换装成功，实际输出约 16.03 秒。
- 客户02 10 秒换装成功。

经验：

- 长版换装可行。
- `target` 仍建议使用 `Clothes`。
- 长版耗时明显更长。

### 3. 背景工作流的一次性长视频能力有限

`VideoRefusion 动态替换视频背景`：

- WebApp ID：`1986353521488523266`
- 客户02 10 秒一次性背景成功。
- 客户01 15/16 秒一次性背景失败。

经验：

- 该工作流不是稳定的任意长视频背景替换工具。
- 10 秒可以尝试一次性。
- 15 秒及以上存在失败风险。
- 如果用户允许分段，分段背景后拼接能绕开失败。
- 如果用户不允许分段，必须换可支持长视频的一次性工作流或标准模型。

### 4. happyhorse 是可探索方向，但需要企业共享 Key

候选标准模型：

- `happyhorse-1.0/video-edit`
- 用途：视频编辑，理论上适合“替换背景、保留人物和动作”。
- API 限制：只允许企业级共享 API Key。

普通 Key 调用报错：

```text
Access Denied: Standard Model API is restricted to Enterprise-Shared API Keys only.
```

后续若拿到企业级共享 API Key，应优先用客户01 15 秒视频测试 happyhorse 一次性背景替换。
